设计要点说明

方案 B 的改进点：

  • 摘要条 — 顶部一行展示 HTTP 方法、路径、状态码、延迟、流式标记，无需滚动即可看到请求概要
  • 全局模式切换 — 一个切换控制所有阶段的「结构化/原始」视图，替代每个区域独立的切换
  • 垂直时间线 — 左侧圆点 + 连线清晰展示请求的四个阶段流向
  • 颜色编码 — 蓝(客户端请求) → 琥珀(代理转发) → 翠绿(LLM响应) → 紫(客户端响应)
  • 行内元数据 — 每个阶段标题旁直接展示关键参数（model、stream、status），无需展开

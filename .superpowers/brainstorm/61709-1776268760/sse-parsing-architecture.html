SSE 解析架构

方案 A：Transform 流 + 实时 SSE 解析

流式管道结构

当前管道 vs 新管道

// 当前

upstream → PassThrough → reply.raw

           ↓ captureChunks[]

           ↓ 日志记录


// 新

upstream → SSEMetricsTransform → PassThrough → reply.raw

               ↓ 实时解析事件

               ↓ 记录 TTFT / 时间戳

               ↓ 提取 usage

               ↓ 流结束后写入 metrics

模块职责划分

sse-parser.ts

职责：通用 SSE 行缓冲解析器

  • 处理跨 chunk 的 SSE 行分割
  • 解析 event: 和 data: 字段
  • 识别 [DONE] 结束标记
  • emit 结构化事件给上层

metrics-extractor.ts

职责：从解析后的事件提取指标

  • 按 api_type 分派到 OpenAI/Anthropic 处理器
  • 记录首个内容 chunk 的时间戳 → TTFT
  • 从 usage 事件提取 token 计数和 cache 信息
  • 流结束后返回完整的 MetricsResult

proxy-core.ts（修改）

职责：集成到现有管道

  • 用 SSEMetricsTransform 替换现有 PassThrough 前段
  • OpenAI 流式请求注入 stream_options
  • 流结束后调用 db.insertMetrics()
  • 非流式：直接从 JSON response 提取

SSE 事件 → 指标映射

Anthropic SSE 事件处理
message_start → 记录 request_start_time input_tokens, cache_*
content_block_delta → 首个有内容的 delta → TTFT 仅首次，后续忽略计时
message_delta → 记录 stream_end_time output_tokens, stop_reason
message_stop → 计算总耗时和 TPS total_duration = end - start
OpenAI SSE 事件处理
首个有 content 的 chunk → TTFT 跳过 role-only chunk
finish_reason chunk → 记录 stream_end_time stop_reason
usage chunk（最后一个） → 提取所有 token 数 需注入 stream_options

非流式请求处理

JSON Response → Metrics

// OpenAI: response.usage

input_tokens ← prompt_tokens

output_tokens ← completion_tokens

cache_read_tokens ← prompt_tokens_details.cached_tokens

stop_reason ← choices[0].finish_reason


// Anthropic: response.usage

input_tokens ← input_tokens + cache_creation + cache_read

output_tokens ← output_tokens

cache_creation_tokens ← cache_creation_input_tokens

cache_read_tokens ← cache_read_input_tokens

stop_reason ← stop_reason


// 非流式：ttft_ms, total_duration_ms, tokens_per_second 均为 NULL

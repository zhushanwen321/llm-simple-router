2. 组件模式：缺乏的 UI 模式

Mockup 引入的模式

  • 折叠面板 — 可折叠的 Headers / Body / Tools 区域
  • 标签切换 — 结构化 vs 原始 JSON/SSE 文本
  • 消息列表 — 带角色头像的消息摘要列表
  • 标签云 — Tools 的 tag pill 展示
  • SSE 事件流 — 逐行展示 SSE 事件类型和摘要
  • 信息横幅 — Claude Code 标识的 indigo 卡片
  • 统计标签 — model/stream/max_tokens 的 pill 展示

当前系统缺失的 Token

  • 无品牌/强调色定义
  • 无角色色彩映射（user/assistant/tool）
  • 无事件类型色彩映射（SSE event types）
  • 无折叠/展开状态样式
  • 无 tag/pill 组件样式
  • 无信息横幅（banner）样式

3. 差距总结

色彩 纯灰阶，无品牌识别 需要引入品牌色 + 角色色 + 事件色
组件 基础 shadcn 组件 需要 Collapsible、Tabs、Tag/Chip
排版 基础 font-sans 需要 mono 字体族用于 JSON/code
密度 标准间距 需要紧凑模式（日志详情信息密度高）
